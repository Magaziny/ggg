# Свадебное интерактивное приглашение (Wedding Invitation Platform)

Современное, интерактивное веб-приложение для свадебных приглашений с возможностью подтверждения присутствия (RSVP), фотогалереей гостей и удобной панелью администратора. 

Проект полностью адаптирован как для локального запуска, так и для деплоя на Serverless-платформу **Vercel** благодаря интеграции с **Turso (libSQL)**.

---

## ✨ Ключевые возможности

- **Интерактивный конверт**: Красивый эффект открытия конверта при первом посещении сайта гостем.
- **Двуязычный интерфейс**: Полная поддержка русского и туркменского языков.
- **RSVP-форма (Подтверждение присутствия)**:
  - Выбор статуса присутствия (приду / не приду).
  - Указание количества гостей и присутствия детей.
  - Выбор предпочтений по напиткам и еде.
  - Указание пищевой аллергии.
  - Сохранение ответов в базу данных.
- **Интерактивная галерея**:
  - Загрузка фотографий гостями прямо во время или после торжества.
  - Возможность ставить лайки к фотографиям.
- **Панель администратора**:
  - Полное управление контентом сайта (имена молодоженов, дата, время, адрес проведения).
  - Настройка тайминга свадебного дня.
  - Настройка цветовой палитры дресс-кода.
  - Список гостей с возможностью экспорта в форматы Excel (`.xlsx`) и скачивания zip-архива с фотографиями.
- **Telegram Уведомления**:
  - Мгновенные уведомления о новых ответах гостей (RSVP) прямо в ваш Telegram-чат через Telegram Bot API (настраивается в панели администратора).

## 🛠️ Технологический стек

- **Фреймворк**: [Next.js 15 (App Router)](https://nextjs.org/)
- **Язык**: TypeScript
- **Стилизация**: Tailwind CSS (с поддержкой плавной смены тем)
- **Анимации**: Framer Motion (плавные переходы, анимация конверта и элементов интерфейса)
- **База данных**: SQLite / Turso (через современный драйвер `@libsql/client`)
- **Иконки**: Lucide React
- **Инструменты экспорта**: adm-zip (архивация фото), xlsx (генерация Excel-отчетов)

## 🚀 Быстрый старт (Локально)

### Требования
Для запуска проекта вам понадобится установленный [Node.js](https://nodejs.org/) версии **20.x** или выше.

### 1. Установка зависимостей
Склонируйте репозиторий и установите все необходимые пакеты:
```bash
npm install
```

### 2. Настройка окружения
Создайте файл `.env` в корневой директории на основе `.env.example`:
```bash
cp .env.example .env
```
Откройте созданный файл `.env` и задайте пароль администратора:
```env
ADMIN_PASSWORD=ваш_надежный_пароль
NODE_ENV=development
PORT=4002
```

### 3. Запуск приложения

#### Режим разработки (Development)
Запустите сервер разработки:
```bash
npm run dev
```
Сайт будет доступен по адресу [http://localhost:4002](http://localhost:4002). Панель администратора доступна по пути `/admin`.

При локальном запуске (если не настроены переменные Turso) драйвер автоматически создаст локальный файл базы данных `db/wedding.db`, настроит необходимые таблицы и заполнит их демонстрационными данными при первом запуске проекта.

#### Быстрый запуск под Windows
Для пользователей ОС Windows в корне проекта подготовлен удобный скрипт `run.bat`. Просто запустите его двойным щелчком мыши: он проверит зависимости, установит их при необходимости и запустит локальный сервер Next.js.

## ☁️ Развертывание на Vercel + База данных Turso

Поскольку Vercel использует Serverless-архитектуру с файловой системой только для чтения (read-only), хранить локальный файл SQLite на сервере невозможно. Для работы базы данных в продакшене используется **Turso** (облачный Serverless SQLite).

### Шаг 1: Создание базы данных в Turso
1. Зарегистрируйтесь на сайте [Turso](https://turso.tech/) (бесплатного тарифа хватит с огромным запасом).
2. Создайте новую базу данных через веб-панель или CLI:
   ```bash
   turso db create wedding-db
   ```
3. Получите URL подключения (Database URL):
   ```bash
   turso db show wedding-db --show-urls
   # Формат: libsql://wedding-db-username.turso.io
   ```
4. Получите токен аутентификации (Auth Token):
   ```bash
   turso db tokens create wedding-db
   ```

### Шаг 2: Деплой проекта на Vercel
1. Создайте новый проект на [Vercel](https://vercel.com/) и импортируйте ваш репозиторий с GitHub.
2. В настройках проекта (Project Settings -> Environment Variables) добавьте следующие переменные окружения:
   - `ADMIN_PASSWORD`: Ваш пароль от панели администратора.
   - `TURSO_DATABASE_URL`: Полученный URL базы данных Turso.
   - `TURSO_AUTH_TOKEN`: Полученный токен аутентификации Turso.
   - `NODE_ENV`: `production`.
3. Нажмите кнопку **Deploy**. 

При первом обращении к развернутому сайту драйвер базы данных автоматически подключится к Turso, создаст все таблицы и запишет начальные настройки.

## 📂 Структура проекта

```text
├── db/                       # Директория с локальной SQLite БД (только для локального режима)
├── public/                   # Статические ресурсы (изображения, шрифты, иконки)
├── src/
│   ├── app/                  # Маршрутизация Next.js (App Router)
│   │   ├── admin/            # Страницы панели администратора
│   │   ├── api/              # API-эндпоинты (RSVP, галерея, настройки)
│   │   └── page.tsx          # Главная страница приглашения
│   ├── components/           # Компоненты интерфейса (конверт, RSVP-форма, галерея и др.)
│   ├── hooks/                # Кастомные React-хуки
│   └── lib/                  # Утилиты, инициализация базы данных и функции-хелперы
├── .env.example              # Шаблон конфигурации окружения
├── run.bat                   # Скрипт быстрого запуска для Windows
├── tailwind.config.ts        # Конфигурация Tailwind CSS
├── tsconfig.json             # Конфигурация TypeScript
└── package.json              # Зависимости и скрипты проекта
```
